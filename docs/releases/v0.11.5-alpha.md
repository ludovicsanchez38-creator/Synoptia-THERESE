# Release v0.11.5-alpha

**Date** : 22 avril 2026
**Mode** : Fast-track (fixes bugs Discord batch 2)
**Version precedente** : v0.11.4-alpha (22 avril 2026, 1h plus tot)

## Contexte

La v0.11.4 corrigeait les 3 regressions du cluster OpenRouter / MCP / Agent. La v0.11.5 attaque les 3 bugs restants du batch Discord S16 :
- #69 Erreurs backend OpenRouter (UnboundLocalError + faux diagnostic Ollama)
- #72 macOS `xattr -cr` au 1er lancement (amelioration ergonomie landing)
- #73 Google Calendar sync KO

Issue #71 (umbrella features cassees) laissee ouverte : j'ai demande plus de details a Dr_logic avant de patcher a l'aveugle.

## Fixes inclus

| Ref | Issue | Type | Description | Fichiers |
|-----|-------|------|-------------|----------|
| B1 | #69 | fix(backend) | `local variable 'e' referenced before assignment` : dans les 3 providers (OpenRouter, Anthropic, Ollama), un `except Exception as e:` imbrique ecrasait la variable `e` de l'outer `except httpx.HTTPStatusError as e:`, provoquant UnboundLocalError a la ligne suivante qui utilisait `e.response.status_code`. Renomme en `body_err`. | `openrouter.py`, `anthropic.py`, `ollama.py` |
| B2 | #69 | fix(backend) | `_default_config()` tombait silencieusement sur Ollama quand l'utilisateur avait choisi un provider cloud sans cle API, d'ou le faux message "Ollama non detecte" alors que le user etait sur OpenRouter. La config retourne desormais le provider choisi avec `api_key=None` (l'erreur API remontee mentionne alors le bon provider). | `llm.py` |
| B3 | #73 | fix(backend) | Google Calendar API recevait des timestamps invalides (`2026-04-22T00:00:00+02:00Z` = offset + Z) quand le datetime etait tz-aware, ce qui renvoyait une liste vide. Nouveau helper `_to_rfc3339_z()` qui convertit en UTC naif avant d'ajouter `Z`. | `calendar_service.py` |
| B4 | #72 | fix(landing) | Section macOS `xattr -cr` enrichie : bouton "Copier" fonctionnel (`navigator.clipboard`), texte clarifie (avant le 1er lancement, obligatoire), ouverture automatique du bloc quand user detecte sur Mac. | `synoptia-site/src/static/therese/alpha/index.html` |

## Tests de regression ajoutes

- `TestBUG69NestedExceptShadowing` : 3 tests verifiant qu'aucun `except Exception as e:` imbrique ne re-apparaisse dans les 3 providers
- `TestBUG69OllamaFallbackRespectsProvider` : 1 test verifiant qu'un `selected_provider=openrouter` sans cle retourne bien OPENROUTER (pas OLLAMA)
- `TestBUG73GoogleCalendarTZ` : 2 tests verifiant la presence du helper `_to_rfc3339_z` + comportement naif OK

Total : 6 nouveaux tests de non-regression (total cumule v0.11.4 + v0.11.5 : 11 tests).

## Decision Claude Code

**MERGER** - 3 fixes backend cibles avec tests, 1 amelioration landing.
- B1, B3 : patches conservateurs (renommage variable, helper ajoute), risque quasi-nul.
- B2 : change un comportement (plus de fallback Ollama silencieux pour provider cloud), mais le test de regression sanctuarise le nouveau comportement et les tests LLM existants passent tous (25/25).
- B4 : landing only, aucun impact backend/frontend app.
