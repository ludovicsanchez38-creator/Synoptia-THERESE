- Inventaire exhaustif des références de modèles (8 emplacements backend + 2 frontend), correction des incohérences pré-existantes (gpt-5.2 vs gpt-5.4, o3-mini vs o4-mini).
- **Recherche web sur sources officielles** (docs.anthropic, platform.openai, ai.google.dev, docs.x.ai, api-docs.deepseek, docs.mistral) : identifiants API exacts, prix, fenêtres de contexte recroisés (anti-hallucination, cutoff de connaissance janvier 2026).

## Vérification

- **CI verte** sur les 4 jobs (Frontend, Tests Backend, Lint Python, Audit sécurité).
- 2 tests de régression alignés sur les modèles actuels (`TestBUG028PricingEUR` : Opus 4.8 / GPT-5.5 au lieu d'Opus 4.6 / GPT-5.2 retirés).

## Note technique (dette / leçon)

- La CI exécute le répertoire de tests **`tests/` à la racine** (1063 tests), pas `src/backend/tests/` (149). Les échecs étaient donc invisibles tant qu'on regardait le mauvais répertoire. À documenter dans `known-patterns.md`.
- Le job backend est parfois tué au teardown (threads orphelins) avant d'imprimer la section FAILURES, masquant le détail des échecs dans les logs CI.
