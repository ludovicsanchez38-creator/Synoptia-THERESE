## Revue adversariale du hotfix (Soso, NO-GO, 11 findings, tous fermés)

- **F1 (bloquant)** : plus JAMAIS d'archive `pre_restore` en clair -
  `_finalize_safety_archive` (succès+passphrase → `.enc` ; échec, legacy ou
  chiffrement KO → suppression, US-003). Tests : legacy sans clair, échec sans clair.
- **F2 (majeur)** : la passphrase de la sauvegarde restaurée chiffre l'état
  antérieur - dit explicitement dans les réponses succès ET échec. Dette
  assumée : pas de `safety_password` distinct (décision produit à prendre).
- **F3 (majeur)** : `delete_all` purge aussi `projects/` ; `files.py` suit
  `THERESE_DATA_DIR`.
- **F4 (bloquant)** : le Board (multi-fournisseurs) exige SON accord
  `llm:board` dans les deux interfaces ; accordable à la confirmation du Board
  ou dans PrivacyTab.
- **F5/F6 (majeur/mineur)** : migration v1 SUPPRIMÉE (l'ancien enregistrement
  était ambigu : images/board/labels) - chaque finalité redemande une fois.
- **F7 (majeur)** : `executeSkill` + `indexFile` exemptés du timeout,
  `initializeAuth` borné à 10 s.
- **F8 (bloquant)** : `prepare_backend_shutdown` avant `/api/shutdown`
  (updater) + exit code 0 = arrêt volontaire ; si l'update échoue après
  l'arrêt, `restart_backend` relance le moteur.
- **F9 (majeur)** : un échec de spawn entre dans le cycle de relance
  (`schedule_sidecar_retry`), état `failed` après 3.
- **F10 (bloquant)** : config Playwright fantôme `src/frontend/playwright.config.ts`
  supprimée (relançait les E2E sur le port réel) + test verrou.
- **F11 (majeur)** : `.session_token` suit `THERESE_DATA_DIR`. Vérifié en réel :
  l'ancien comportement avait bien effacé le jeton du profil de Ludo pendant
  les E2E ; plus reproduit après fix.
