## Contenu — PR #84 (quick-wins audit Mail/Calendrier du 18/06)

Issu de l'audit Mail/Calendrier (panel 3 experts). Deux quick-wins, développés en TDD.

| Quick-win | Type | Description |
|---|---|---|
| Résumé de fil IA local (`summarize_emails`) | `feat` | Outil chat : « résume-moi le fil avec X / mes mails non lus ». Lit les messages, condense, résume via le LLM configuré. 100% local. |
| Plage horaire d'agenda adaptative | `fix` | Vues Semaine/Jour : la plage horaire s'élargit pour englober les RDV tôt/tard (avant : codée en dur 8h-20h / 6h-22h, les RDV hors plage disparaissaient). |

## Audit 3 agents (avant merge)

| Agent | Verdict | Suite donnée |
|---|---|---|
| Sécurité | WARNING | **Corrigé** : `_summarize_emails` encapsule le contenu des emails dans des délimiteurs `[Source: email]` + neutralisation (`sanitize_for_context`) + system prompt durci (anti-injection de prompt). Plafond 24000c sur le condensé (garde-fou contexte LLM). + test anti-injection. |
| UX/UI | WARNING | **Corrigé** : 2 régressions critiques — le scroll des vues Semaine/Jour utilisait les heures statiques au lieu de la plage dynamique → déplacé après le calcul de plage + dépend de `weekStartHour`/`dayStartHour`. |
| Avocat du diable | (échec format) | Cas limites couverts par les tests unitaires `getVisibleHourRange` (multi-jours, minuit, datetime invalide, all_day) + garde-fou volume du résumé. |

**Décision arbitre** : MERGER après corrections (faites). Les 2 WARNING légitimes provenaient du code de la PR, corrigés et re-testés.
