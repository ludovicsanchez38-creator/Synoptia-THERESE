## Contenu

Déclencheur : retours du testeur **capov** (« Richard ») sur le Discord après la 0.20.

### Correctifs (régressions 0.20) — branche `fix/bugs-post-0.20`
- **Calendrier fuseau horaire** : `timeZone` codé en dur `Europe/Paris` côté backend (create) ou absent (update) → un événement saisi à Toronto s'affichait avec 6 h de décalage. Le front transmet désormais le fuseau IANA réel (`Intl…resolvedOptions().timeZone`), utilisé pour create et update. 2 tests de régression backend.
- **Calendrier validation** : message explicite si la fin précède le début (comparaison sur l'heure murale, robuste au changement d'heure / DST).
- **BUG-102 mail vide + réponse sans historique** : `EmailDetail` n'appelait jamais `getEmailMessage`, et la liste backend charge en `format='metadata'` (sans corps) → corps chargé à l'ouverture + réponse citée. Test Vitest.
- **BUG-105 bouton Répondre rogné** : barre d'actions passée en `flex-wrap`.
- **BUG-104 « Projet » ouvrait les Contacts** : la refonte nav 0.20 avait orpheliné `ProjectsKanban` → nouvelle vue dédiée `projects` (`ProjectsPanel`, kanban par statut, glisser-déposer).

### Nouvelle vue Accueil — branche `feat/dashboard-accueil`
Le dashboard « Ma journée » (inaccessible une fois fermé, vide-donc-muet, truffé de faux : recherche morte, badge « Opus 4.8 » codé en dur, fausse barre de saisie) est remplacé par une **vue `home` persistante hybride** : lanceur (actions rapides + conversations récentes) + journée (agenda + à traiter), bande « Mise en route » qui se retire, badge fournisseur réel, accès par bouton header + ⌘K `home.open`. Endpoint `GET /api/dashboard/setup-status`.
