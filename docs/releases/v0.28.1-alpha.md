## Revue adversariale (avocat du diable)

Un sous-agent adversarial « expert coding » a audité l'ensemble des corrections avant tag.

- **1 BLOQUANT confirmé** (et corrigé) : `_get_email_provider` / `_get_calendar_provider` réassignaient `account` au retour de `ensure_valid_access_token` (token déchiffré, str) puis lisaient `account.access_token` → `AttributeError`. Outils chat email + calendrier non fonctionnels pour tout compte Google. Pré-existant, révélé car BUG-133 réécrivait cette fonction ; branche gmail sans aucun test. Corrigé + 2 tests de régression.
- **2 mineurs corrigés** : notification si fichier de skill introuvable au téléchargement ; sélection déterministe du calendrier local.
- **Axes jugés sains** (vérifiés) : cycle de vie de la session dans le flux SSE (`expire_on_commit=False`), migration ad-hoc `extra_data` (idempotente), parsing `extra_data` malformé, regex de masquage du code.

Au passage : un `I001` ruff (tri d'imports) dans un test de BUG-133 aurait mis la CI ruff en rouge → corrigé.
