## Changements fonctionnels

### BUG-057 : Annulation délibération Board
- **Problème** : En mode souverain avec des modèles Ollama volumineux, l'app lag sans possibilité d'arrêter
- **Fix** : AbortController + bouton "Annuler" rouge dans la vue délibération
- **Bonus** : Indicateurs de taille des modèles (Go/Mo) + recommandation hardware (vert < 4 Go, jaune 4-8 Go, rouge > 8 Go)
- **Fichiers** : `BoardPanel.tsx`, `DeliberationView.tsx`, `AdvisorArcLayout.tsx`, `board.ts`

### BUG-058 : Board cloud vide (5 conseillers sans réponse)
- **Problème** : BUG-052 (modèle Ollama respecté) envoyait le `llm_model` utilisateur (ex: "claude-opus-4-6") à TOUS les providers (GPT, Gemini, Grok, Mistral) → crash silencieux
- **Fix** : `get_llm_service_for_provider()` ne lit le modèle DB que si le provider demandé correspond au provider principal de l'utilisateur
- **Fichier** : `llm.py` (services)

### BUG-059 : Email re-auth bloqué sur "En attente d'autorisation"
- **Problème** : Le polling ne détectait que les nouveaux comptes (count++), pas les comptes re-autorisés (même count, updated_at changé)
- **Fix** : Stockage snapshot initial `(id → updated_at)` + comparaison des timestamps. Backend : ajout `updated_at` dans la réponse auth status
- **Fichiers** : `VerifyStep.tsx`, `email.ts` (frontend), `email.py` (backend router)
