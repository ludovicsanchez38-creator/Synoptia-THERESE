## Fonctionnel

Nouveau réglage « Effort de raisonnement » (Paramètres > IA, sous le choix du
modèle) : Auto / Faible / Moyen / Élevé / Maximal, persisté entre sessions
(préférence `llm_effort`), exposé par `GET/POST /api/config/llm` (champ `effort`,
`auto` = effacer). Traduction par provider, envoyée UNIQUEMENT aux modèles au
support vérifié (sources du 10/07/2026) :

- **Anthropic** `output_config.effort` low→max : Fable 5, Sonnet 5, Sonnet 4.6,
  Opus 4.5+ (préfixes `_EFFORT_PREFIXES`). Pas Haiku (l'API le refuse).
- **OpenAI** `reasoning_effort` : GPT-5.6 uniquement (5.5/5.4 non sourcés → rien
  d'envoyé, dette documentée).
- **xAI** `reasoning_effort` low/medium/high : grok-4.5, `max` plafonné à `high`.
- **Ollama** `think` low/medium/high (`max` plafonné, non standardisé - gpt-oss
  le refuse) ; modèles non-thinking : détection « does not support thinking » +
  retry silencieux SANS think (sentinel `_THINK_UNSUPPORTED`, pattern dégradation
  tools US-009).
- **Auto** (défaut) : rien d'envoyé, comportement identique aux versions
  précédentes. Aucun autre provider (Gemini, Mistral, DeepSeek, OpenRouter,
  Perplexity, Infomaniak) ne reçoit le champ.
