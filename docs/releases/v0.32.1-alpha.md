## Détail des fixes (racines confirmées avant code)

- **BUG-122 (rouvert par capture Dr_logic)** : STARRED absent de la table de
  résolution des dossiers IMAP (ce n'est pas un dossier mais le flag
  `\Flagged`) -> repli silencieux sur l'INBOX non filtrée sous « Favoris ».
  Fix : `flagged_only` sur `list_messages` (critère `AND(flagged=True)`),
  routage STARRED dédié dans `_list_messages_imap`.
- **BUG-136** : `_generate_document` (outil workspace, appelable N fois par
  tour) n'émettait JAMAIS `skill_file` -> fichiers sans carte ; et le message
  ne persistait qu'UN fichier. Fix : collecteur par tour (ContextVar) drainé
  avant `done` (un événement par fichier), persistance
  `extra_data.skill_files` (liste, lecture legacy `skill_file` conservée),
  cartes multiples dans MessageBubble. Le volet « fichiers créés par des apps
  externes » (PDF exporté par Word dans outputs) = suggestion observateur de
  dossier, hors scope de ce fix.
- **BUG-137** : `detect_skill_intent` (mots-clés type « créer.*document »)
  forçait une génération sur des phrases ordinaires. Fix : détection par
  mots-clés SUPPRIMÉE de `resolve_skill_from_message` (chemins explicites
  conservés : `{action: produire}`, `{{action: skill}}`, sélecteur UI) +
  consigne de l'outil `generate_document` resserrée (demande explicite,
  doute = demander). Règle produit posée avec Dr_logic dans #bugs.
