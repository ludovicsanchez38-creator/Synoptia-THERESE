# Release v0.11.0-alpha

**Date** : 2026-04-07
**Mode** : Fast-track (code audité par UltraJury en session)
**Version précédente** : v0.10.8

## Ce qui change (6 features, 3 sprints)

### Sprint 1 - Quick Wins

**S13-5 : Refonte PDF**
- Nouveau système `PDFTheme` (dataclass réutilisable) avec palette Synoptïa
- Factures avec header coloré, tableau stylisé (header bleu, zebra striping), bloc totaux
- Footer avec mentions légales et numéro de page
- Compatible facture, devis et avoir

**S13-2 : Import ICS + VCard**
- `POST /api/calendar/import-ics` : corrigé bug `cal.name` -> `cal.summary`
- `GET /api/calendar/export-ics` : export .ics avec attendees, récurrence, all-day
- `POST /api/memory/contacts/import` : import VCF avec détection doublons (email + nom)
- `GET /api/memory/contacts/export` : export VCF complet
- Boutons Import/Export dans CalendarPanel et MemoryPanel

**S13-6 : Bibliothèque de prompts**
- 34 prompts en français répartis en 6 catégories (Email, Commercial, Admin, Rédaction, Organisation, Juridique)
- `GET /api/prompts/library` et `/search?q=xxx`
- Composant `PromptLibrary.tsx` avec accordéons, recherche, placeholders surlignés
- Intégré dans HomeCommands et CommandPalette

### Sprint 2 - Fondations

**S13-4 : Modes UX Standard / Contributeur**
- Toggle dans les paramètres, persisté localStorage
- Mode Standard masque : MCP/Outils, Board, Agents, onglets avancés Settings
- Mode Contributeur : tout visible (défaut pour les alpha testeurs)

**S13-3 : Module devis**
- Type "devis" ajouté au modèle Invoice
- Statuts spécifiques : draft, sent, accepted, refused, expired, converted
- Champ `validite_jours` (défaut 30 jours)
- Endpoint `PATCH /{id}/devis-status` pour accepter/refuser/expirer
- Conversion devis → facture
- Filtres et badges colorés dans l'UI
- Migration Alembic `validite_jours`
- 15 tests dédiés

### Sprint 3 - Différenciation

**S14-2 : Actions agents spécialisés**
- Système `ActionAgent` + `ActionRunner` pour tâches multi-étapes
- 6 agents prédéfinis : rapport-hebdo, relance-clients, prep-rdv, audit-tresorerie, veille-concurrent, onboarding-client
- Endpoints : `GET /api/actions`, `POST /api/actions/{id}/run`, `GET/DELETE /api/actions/tasks/{id}`
- `ActionPanel.tsx` avec progression temps réel, annulation
- Détection `{{action: xxx}}` dans le chat
- Intégré dans CommandPalette et ChatHeader
- 28 tests dédiés

## Résumé technique

- **Version** : 0.10.8 → 0.11.0
- **Fichiers** : ~35 créés/modifiés, ~4000 lignes ajoutées
- **Tests** : 796 → 839 (+43 nouveaux, 0 régression)
- **3 échecs préexistants** : z-index email, localhost hardcodé (non liés)
