## Nouvelles fonctionnalites (V3)

### 1. File d'attente chat

**Probleme** : Pendant que THERESE repond en streaming, l'utilisateur ne peut pas preparer son prochain message. Il doit attendre la fin de la reponse, ce qui casse le flux de pensee.

**Solution** : Pendant le streaming, l'utilisateur peut taper 1 prompt supplementaire. Le message apparait dans une bulle italique sous la reponse en cours. Des que le streaming finit, le prompt en file est envoye automatiquement.

**Fichiers** :
- `src/frontend/src/stores/chatStore.ts` - nouvel etat `queuedPrompt`
- `src/frontend/src/components/chat/ChatInput.tsx` - logique de file d'attente, bulle UI italique, envoi automatique

### 2. Outils installes (backend)

**Probleme** : Chaque appel a un "outil" LLM regenere le script Python a chaque fois, meme pour des operations identiques. Couteux en tokens et lent.

**Solution** : Infrastructure pour installer des scripts Python valides dans `~/.therese/tools/`. Le LLM genere un script une seule fois, THERESE le valide (sandbox + test), puis l'installe. Les appels suivants utilisent directement le script installe.

**Nouveaux fichiers** :
- `src/backend/app/models/installed_tool.py` - modele SQLModel pour les outils installes
- `src/backend/app/services/tool_installer.py` - service de validation et installation
- `src/backend/app/routers/tools.py` - endpoints API (CRUD outils)

**Fichiers modifies** :
- `src/backend/app/services/registry.py` - `discover_installed_tools()` pour charger les outils au demarrage
- `src/backend/app/main.py` - montage du router `/tools`

### 3. Commandes V3 unifiees

**Probleme** : Le systeme de commandes existant est fragmente entre commandes systeme et commandes utilisateur, sans schema de validation commun.

**Solution** : Refonte complete avec un JSON schema unifie. Les commandes utilisateur custom suivent le meme format que les commandes systeme.

**Nouveaux fichiers** :
- `src/backend/app/models/schemas_commands.py` - schemas Pydantic pour les commandes V3
- `src/backend/app/services/commands_v3.py` - service de gestion des commandes
- `src/backend/app/routers/commands_v3.py` - endpoints API commandes V3

### 4. Brave Search

**Probleme** : THERESE n'a aucun acces au web pour enrichir ses reponses avec des informations actuelles.

**Solution** : Integration de l'API Brave Search. L'utilisateur configure sa cle API dans les parametres, et THERESE peut effectuer des recherches web.

**Fichiers** :
- `src/backend/app/services/web_search.py` - service Brave Search
- `src/frontend/src/components/settings/SettingsModal.tsx` - UI configuration cle Brave
- Endpoint `has_brave_key` dans la config backend

### 5. OpenRouter ameliore

Meilleure prise en charge des modeles OpenRouter : gestion des prefixes provider, compatibilite elargie avec les modeles disponibles.

### 6. Sonnet 4.6 rename

Mise a jour de toutes les references `claude-sonnet-4-5` vers `claude-sonnet-4-6` suite au renommage officiel Anthropic.
