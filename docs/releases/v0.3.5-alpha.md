## Changements

### BUG-050 : Backup fichier clé Fernet (critique)

**Problème** : Le changement de `productName` de "THÉRÈSE" à "THERESE" en v0.3.4 a modifié la signature du binaire macOS. Le Keychain macOS bloque l'accès de la nouvelle signature à l'ancienne clé Fernet. Le backend génère alors une NOUVELLE clé, rendant toutes les données chiffrées (clés API, profil utilisateur) illisibles.

**Fix** (`src/backend/app/services/encryption.py`) :

1. **Backup fichier systématique** : `_create_key_in_keychain()` écrit aussi dans `~/.therese/.encryption_key` (permissions 0o600) via la nouvelle méthode `_write_key_backup()`
2. **Détection d'incohérence** : `_get_or_create_key()` compare la clé Keychain avec le fichier backup. Si elles diffèrent → la clé Keychain est récente (mauvaise), la clé fichier est l'originale (bonne)
3. **Restauration automatique** : si incohérence détectée, la clé fichier est restaurée dans le Keychain
4. **Migration non-destructive** : `_migrate_key_to_keychain()` ne supprime plus le fichier source

**Résultat** : la clé de chiffrement survit aux changements de signature binaire (changement de nom, recompilation, mise à jour Tauri).
