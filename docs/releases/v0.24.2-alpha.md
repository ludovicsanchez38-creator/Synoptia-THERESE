# Release THÉRÈSE v0.24.2-alpha

**Date** : 11/06/2026
**Type** : PATCH (4 correctifs de bugs)
**Mode** : fast-track (pas de PR Zézette ; correctifs commités directement sur `main` par Mac Claude, en TDD)

## Contexte

4 bugs signalés par Ludo depuis son app installée (0.24.1), corrigés et poussés
sur `main` avant la release. Aucune PR ouverte à merger.

## Correctifs inclus

| Commit | Scope | Bug | Cause racine | Correctif |
|--------|-------|-----|--------------|-----------|
| `7e0bfd0` | images | Génération d'images « clé non configurée » (GPT/Gemini/Fal) malgré clés enregistrées | `_get_api_key_from_db` lisait la base via une connexion SQLite **standard**, illisible sur la base chiffrée SQLCipher (US-014). Exception avalée → `None`. Le **test** de la clé passait (connecteur applicatif) mais la **génération** échouait. | Lecture via `db_connect` (pose la clé SQLCipher). |
| `e8525a3` | gemini | Chat Gemini `Invalid function name` → 400 sur toute la requête, chat muet | Les noms d'outils MCP (`serveur__outil`, `/`, espaces, chiffre en tête) partaient bruts dans `functionDeclarations[].name`. Seuls les `parameters` étaient sanitisés, pas le `name`. | Sanitisation conforme (lettre/`_` en tête, `[A-Za-z0-9_.:-]`, 128 max) + table nom-sanitisé→nom-réel pour ré-associer le `functionCall` renvoyé + re-sanitisation cohérente du tour rejoué en continuation. |
| `83564ad` | email | Bouton « Générer une réponse » coupé en bas | Barre d'actions sans `shrink-0` : le corps du mail (`flex-1`) la poussait hors du cadre `overflow-hidden`. | `shrink-0` sur la barre d'actions et le bloc d'erreur. |
| `917200f` | prompts | Bibliothèque de prompts : 1re carte coupée en haut | Conteneur de l'overlay sans hauteur définie (`max-h-[85vh]` seul) alors que `PromptLibrary` est en `h-full` + scroll interne. | Hauteur fixe `h-[85vh]` + `overflow-hidden` (scroll géré dans `PromptLibrary`). |

## Décision (arbitrage Claude Code)

**MERGER / RELEASER.** Les 2 correctifs backend sont prouvés par des tests de
régression écrits en TDD (RED→GREEN). Les 2 correctifs UI sont des ajustements
de layout Tailwind ciblés, cause racine lue dans le code et chaîne de layout
parente vérifiée, validés par tsc + eslint + Vitest. Aucun risque identifié.
