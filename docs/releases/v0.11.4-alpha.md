# Release v0.11.4-alpha

**Date** : 22 avril 2026
**Mode** : Fast-track (fixes regressions v0.11.3)
**Version precedente** : v0.11.3-alpha (13 avril 2026)

## Contexte

La release v0.11.3 du 13 avril avait integre 11 fixes pour Dr_logic-3D (commit 9d14a4d). Entre le 14 et le 17 avril, Dr_logic a remonte 3 regressions ou fix incomplets post-v0.11.3 sur Discord #bugs. La v0.11.4 cible exclusivement ces regressions.

Issues GitHub liees : #68 (R1+R2), #70 (R3).

## Fixes inclus

| Ref | Type | Description | Fichiers |
|-----|------|-------------|----------|
| R1 | fix(frontend) | Modele OpenRouter perso disparait au reload : `resolveModelForProvider` ignorait `llmConfig.available_models` et retombait sur la liste statique `PROVIDERS`. Fusion des deux listes (dedupliquee). | `modelResolution.ts`, `SettingsModal.tsx` |
| R2 | fix(frontend) | Onglet Agent ne permettait pas de creer un modele custom (affichait seulement les customs deja sauves). Ajout d'un bouton "Personnalise" + input libre (pattern LLMTab). Composant factorise `AgentModelSelect`. | `AgentsTab.tsx` |
| R3 | fix(backend) | Installation MCP bloquante sous Windows : `check_preset_requirements()` et `install_preset()` utilisaient une logique PATH Unix-only (separateur `:`, pas de chemins Windows) alors que `start_server()` etait Windows-aware. Factorisation d'un helper `build_mcp_enriched_path()` partage par les 3 endpoints. Utilise `os.pathsep`, inclut `%PROGRAMFILES%\nodejs`, `%APPDATA%\npm`, `%LOCALAPPDATA%\fnm`, `~/.volta/bin`. L'endpoint `check-requirements` expose desormais `platform` et `enriched_path` pour faciliter le diagnostic. | `mcp_service.py`, `mcp.py` |

## Tests de regression ajoutes

  - inclut les chemins Windows (`nodejs`, `npm`) quand `sys.platform == "win32"`
