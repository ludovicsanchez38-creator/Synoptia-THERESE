## Revue Soso (unique, fin de lot) : NO-GO -> remédiation -> contre-vérification

NO-GO initial, 11 findings. Contre-vérification de ma part : 9 confirmés, 2
requalifiés. Remédiation en 2 vagues (commits fc3b948 + suivant) :

- **F1 (majeur, le vrai trou)** : le schéma backend refusait début = fin même
  en journée entière - l'UI acceptait, l'API 422. Corrigé + prouvé de bout en
  bout (événement créé via l'UI réelle, présent en base).
- **F2** : les routes calendrier Google passent par CalendarService brut, pas
  par le provider converti -> conversions appliquées aux sites des routes.
- **F4** : import/export ICS convertis (DTEND exclusif RFC 5545) + **migration
  ad-hoc unique** (marqueur `preferences.allday_end_semantics`) : le cache
  Google/CalDAV pré-0.41.1 stockait des fins exclusives, converties -1 jour
  clampé au premier démarrage ; calendriers locaux intacts, idempotente.
- **F5** : ensureDefaultCalendar déduplique les appels en vol.
- **F6** : navigation slash directe refusée pendant un streaming (état vivant).
- **F7** : descriptions durcies - send_email annonce qu'il ne peut PAS joindre
  de document ; search_invoices interdit de prétendre à l'envoi. Garde runtime
  (bloquer send_email post-search) volontairement non ajoutée : dette volet 2.
- **F8** : métacaractères ILIKE échappés (\\, %, _) + escape explicite.
- **F9 + N1** : sanitize_for_context neutralise désormais les marqueurs forgés
  ([Source:/[End) DANS les données - un nom de contact ne peut plus sortir de
  l'enveloppe. Profite à tous les consommateurs (emails compris).
- **F10 + N2** : mapping micro sur le TEXTE de l'erreur (chaînes du plugin
  Tauri, dont « No default input device available ») + garde anti-faux
  diagnostic (une permission de FICHIER n'est pas déguisée en permission micro).
- **F3 requalifié préexistant** (accepté par la contre-vérification Soso) : la
  grille et le dashboard n'indexent les événements toute la journée que sur
  start_date - un multi-jour ne s'affiche que sur son 1er jour, DEPUIS TOUJOURS.
  Dette : développer la plage sur chaque jour couvert (grille, vue jour,
  dashboard).
- **F11 requalifié dette dev** (accepté) : browser_tool_available vérifie le
  module python, pas le chromium téléchargé - n'affecte que les postes de dev
  avec l'extra e2e sans `playwright install`.
